## Comment Tags Convention

### TODO
Used to mark something that still needs to be implemented or completed.
Example:
# TODO: add validation for negative values

### FIXME
Indicates a known bug or incorrect behavior that must be fixed.
Example:
# FIXME: division by zero not handled

### NOTE
Adds important context or explains a non-obvious decision.
Example:
# NOTE: input kept as string to detect int vs float later

### HACK
Temporary or non-ideal solution used to unblock progress.
Example:
# HACK: quick workaround until proper parser is implemented

### WARNING
Highlights risky behavior or assumptions that could cause issues.
Example:
# WARNING: assumes trusted input

### OPTIMIZE
Marks code that works but could be improved in performance or design.
Example:
# OPTIMIZE: avoid repeated conversions

### REVIEW
Signals code that should be revisited or reconsidered later.
Example:
# REVIEW: check if this branching is necessary
